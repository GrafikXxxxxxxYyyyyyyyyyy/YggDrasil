# Scheme — полная схема системы Иггдрасиль (новый канон)

**Назначение:** единый документ **архитектуры всей системы** — от фундамента до вселенной. Scheme задаёт *как* всё устроено: конфигурация, уровни, цикл мира (три стадии), state, контракты и связь с документацией. Он не есть сама реализация (код/рантайм), а **схема материализации**, на которой строится канон и код.

**Философия — якорь.** Документы в [../philosophy/](../philosophy/) — единственный идеологический ориентир. Scheme и канон должны ей соответствовать.

**Язык:** русский.

В текущей философии и новом каноне используется понятие **переход** и **триединство** (два начала + переход); понятие «синтез» не входит в каноническую формулировку.

**Детальная техническая документация по уровням:** [docs/](docs/) — 01_FOUNDATION … 07_UNIVERSE; тематические документы: сериализация, глоссарий.

## 1. Основание: два начала и фундамент

Вся система держится на **двух началах** и их **переходе** ([../philosophy/Philosophy-Part2.md](../philosophy/Philosophy-Part2.md)).

| Начало | Сущность в проекте | Смысл |
|--------|--------------------|--------|
| **Материальное** | **Abstract Base Block** (блок) | Хранит и несёт: параметры, тензоры, конфигурацию, состояние, веса. **Атом сборки.** |
| **Идеальное** | **Abstract Graph Node** (узел гиперграфа) | Задаёт **место** и **связи** (гиперрёбра). Не хранит содержание — только положение и связи. |

**Правило перехода («блок → узел»):** реализуется **узлами-задачами** — классами с **двойным наследованием** (Block + Node). Один объект = блок (данные, вычисление forward) и узел (положение, связи, объявление портов). **Гиперграф задачи** собирается **только из таких объектов**; обёртки «узел с блоком внутри» нет. По гиперрёбрам передаётся поток (данные, state, обмен).

**Контракт блока:** идентификация (block_type, block_id), выполнение **forward**(входы) → выходы, state_dict; порты не объявляет. **Контракт узла:** node_id, объявление портов (declare_ports), get_input_ports/get_output_ports, run(inputs)→forward(inputs). Спецификация: [docs/01_FOUNDATION.md](docs/01_FOUNDATION.md).

**Над фундаментом:** абстрактные узлы-задачи (Backbone, Injector, Conjector, Inner Module, Outer Module, Helper, Converter) — **двойное наследование** Block и Node; только они являются вершинами гиперграфа задачи. Спецификация: [docs/02_ABSTRACT_TASK_NODES.md](docs/02_ABSTRACT_TASK_NODES.md).

## 2. Иерархия уровней: общая схема

На каждом уровне структура — **гиперграф**; различается **что является узлом** и **что течёт по связям**.

| Уровень | Узлы гиперграфа | Что течёт по связям | Документ |
|---------|------------------|---------------------|----------|
| **Фундамент** | Узлы-задачи (Block+Node в одном объекте) | — | [docs/01_FOUNDATION.md](docs/01_FOUNDATION.md) |
| **Абстрактные узлы-задачи** | Backbone, Injector, Conjector, … | — | [docs/02_ABSTRACT_TASK_NODES.md](docs/02_ABSTRACT_TASK_NODES.md) |
| **Гиперграф задачи** | Экземпляры узлов-задач | **Данные** между портами | [docs/03_TASK_HYPERGRAPH.md](docs/03_TASK_HYPERGRAPH.md) |
| **Воркфлоу** | Целые гиперграфы задач | **Данные** между гиперграфами | [docs/04_WORKFLOW.md](docs/04_WORKFLOW.md) |
| **Стадия** | Воркфлоу | Внутри стадии: **данные**; вход/выход стадии: **state** | [docs/05_STAGE.md](docs/05_STAGE.md) |
| **Мир** | Стадии | **State** (контейнер слотов) | [docs/06_WORLD.md](docs/06_WORLD.md) |
| **Вселенная** | Миры | **Обмен** (payload_spec) | [docs/07_UNIVERSE.md](docs/07_UNIVERSE.md) |

**Единая форма:** на всех исполняемых уровнях — гиперграф, порты (или их аналог), **run**, контракт. Меняется только масштаб (узел = задача, воркфлоу, стадия, мир). **Движок гиперграфа** (планирование, буферы, исполнитель) применяется на уровнях 0.3–0.7; на уровне стадии и выше по связям между узлами уровня течёт state или обмен, а не «сырые» данные между портами. Подробно: [docs/HYPERGRAPH_ENGINE.md](docs/HYPERGRAPH_ENGINE.md).

## 3. Контракты run по уровням

| Уровень | Сигнатура run | Вход | Выход |
|---------|----------------|------|--------|
| **Гиперграф задачи** | run(hypergraph, inputs, **options) | inputs — словарь по внешним входам | outputs — словарь по внешним выходам |
| **Воркфлоу** | run(workflow, inputs, **options) | inputs — по внешним входам воркфлоу | outputs — по внешним выходам |
| **Стадия** | run(stage, state, context?, **options) | state (контейнер слотов), опционально context | state (обновлённый) |
| **Мир** | run(world, state?, action?, context?, **options) | state (опционально), action (опционально), context | state (или результат витка) |
| **Вселенная** | run(universe, world_inputs?, **options) | world_inputs — словарь world_id → входы для мира | results (по соглашению реализации) |

Сериализация на всех уровнях: **конфиг** (структура) + **чекпоинт** (веса, state_dict); загрузка из чекпоинта на каждом уровне. Подробно: [docs/SERIALIZATION.md](docs/SERIALIZATION.md).

## 4. Уровень гиперграфа задачи и воркфлоу

**Гиперграф задачи** — **первый исполняемый уровень**. Один гиперграф = **одна цельная задача** (text-to-image, апскейл, агентный сценарий и т.д.). Узлы — экземпляры узлов-задач; по гиперрёбрам — данные между портами. Поддержка циклов и итеративных схем. [docs/03_TASK_HYPERGRAPH.md](docs/03_TASK_HYPERGRAPH.md).

**Воркфлоу** — **второй исполняемый уровень**. Гиперграф, узлы которого — **целые гиперграфы задач**. По гиперрёбрам — данные между выходами одного гиперграфа и входами другого. Циклы между гиперграфами поддерживаются. Комбинированные сценарии (цепочка или ветвление задач). [docs/04_WORKFLOW.md](docs/04_WORKFLOW.md).

---

## 5. Уровень стадии: метаморфоза state

**Стадия** — **третий исполняемый уровень**. Гиперграф, узлы которого — **воркфлоу**. Внутри стадии между воркфлоу передаются **данные**; **внешний** вход и выход стадии в контексте мира — **state** (контейнер слотов). Одна стадия = **одна метаморфоза** state (материя → идея, идея → материя). Маппинг: state_input_map, state_output_map; условие can_run(stage, state). [docs/05_STAGE.md](docs/05_STAGE.md).

---

## 6. Уровень мира: цикл, state, три стадии

**Мир** — **четвёртый исполняемый уровень**. Гиперграф, узлы которого — **стадии**. По гиперрёбрам передаётся **state**. Порядок обхода **фиксирован**, граф **цикличен**.

**Цикл мира (три стадии):**

```
  автор  →  среда  →  творец  →  автор  →  …
```

- **Стадия автора:** материальные артефакты → идея (описание того, что произошло в мире).
- **Стадия среды:** переход; принимает описания от автора, помещает в историю мира; **обновление контента мира** при полном state; **порождение нового state** (развитие мира) для следующего витка; опционально **action** пользователя.
- **Стадия творца:** идея (намерение, конфигурация) → артефакты.

**State** — контейнер с именованными слотами; схема задаётся **state_schema**. Условия выполнения стадий — **can_run(stage, state)**. **Первая итерация:** мир может начаться без предзаданного state (развитие в стадии среды порождает первый state). [docs/06_WORLD.md](docs/06_WORLD.md).

---

## 7. Уровень вселенной

**Вселенная** — **пятый (последний) исполняемый уровень** и **высший уровень онтологии**. Гиперграф, узлы которого — **миры** (развивающиеся или End World). По гиперрёбрам — **обмен** (payload_spec: state snapshot, артефакты, контекст, события, сущности). **Эфир вселенной** — единая среда (общий контекст, поток трансляции). **Переход сущностей** между мирами (origin_world_id, кросс-мировое влияние). Уровень предназначен для **сообщества**: сравнение миров, взаимодействие, слияние миров. **End World** — мир только для чтения. [docs/07_UNIVERSE.md](docs/07_UNIVERSE.md).

---

## 8. Граница онтологии и расширяемость

**Выше вселенной уровня нет.** Метавселенная как формальный уровень **не вводится**; «мета» — взаимодействие пользователей в реальном мире и публикация вселенных на единой площадке.

**Расширяемость:** новые блоки, узлы-задачи, миры вводятся **без изменений движка** — реализация контракта (порты, run, сериализация) и **регистрация в реестре**. Реестр типов блоков: [docs/01_FOUNDATION.md](docs/01_FOUNDATION.md) §7.

## 9. Сводка: быстрый ориентир

| Вопрос | Ответ |
|--------|--------|
| **Основание?** | Два начала: блок (материя), узел (идея). Правило: блок → узел. Гиперграф — форма целого. |
| **Уровни?** | Гиперграф задачи → Воркфлоу → Стадия → Мир → Вселенная. На каждом — гиперграф; по связям — данные, state или обмен. |
| **Мир?** | Цикл из трёх стадий: автор → среда → творец. По рёбрам — state (контейнер слотов). Мир может развиваться сам; action опционален. |
| **State?** | Контейнер слотов; state_schema задаёт слоты и семантику. World update при полном state; Development порождает новый state. |
| **Вселенная?** | Гиперграф миров; эфир; обмен по payload_spec; сущности, End World, сообщество. Высший уровень. |
| **Якорь?** | Философия (Part1, Part2, Part3) — неизменяемый ориентир. Scheme и канон строятся по философии. |

**Термины и ссылки:** [docs/GLOSSARY.md](docs/GLOSSARY.md).
