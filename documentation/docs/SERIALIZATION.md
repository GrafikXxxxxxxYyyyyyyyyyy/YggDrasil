# Сериализация на всех уровнях

**Назначение:** единый технический документ по **сериализации и загрузке** на каждом уровне системы Иггдрасиль — от блока до вселенной. Задаёт принцип «конфиг + чекпоинт» на каждом уровне, загрузку из чекпоинта, особые случаи (мир — два поля, вселенная — агрегат чекпоинтов миров, End World). Канонический источник для реализации save/load и воспроизводимости.

**Связь с уровнями:** [01_FOUNDATION.md](01_FOUNDATION.md) §9 (блок, узел, реестр), [03_TASK_HYPERGRAPH.md](03_TASK_HYPERGRAPH.md) §9, [04_WORKFLOW.md](04_WORKFLOW.md) §8, [05_STAGE.md](05_STAGE.md), [06_WORLD.md](06_WORLD.md) §9, [07_UNIVERSE.md](07_UNIVERSE.md) §11.

**Язык:** русский.

---

## 1. Принцип

На **каждом уровне** сущность системы должна быть **полностью сериализуема** и **восстанавливаема** из сохранённого состояния:

- **Структура** задаётся **конфигом** (узлы, рёбра, маппинги, порядок, метаданные).
- **Исполняемое состояние** (веса, параметры, кэш) задаётся **чекпоинтом** (state_dict и при необходимости дополнительные поля).

**Сохранить** = записать конфиг + чекпоинт. **Загрузить** = восстановить сущность из конфига и загрузить чекпоинт в соответствующие блоки/узлы. При одинаковых конфиге, чекпоинте и входах поведение **run** воспроизводимо в пределах контракта блоков и движка.

## 2. Уровни и что сериализуется

| Уровень | Сущность | Конфиг (структура) | Чекпоинт (веса/состояние) |
|---------|----------|---------------------|----------------------------|
| **Фундамент** | Block, Node | block_type, block_id, конфиг блока, порты; узел = имя + ссылка на блок + связи. | **state_dict** блока. Загрузка блока из чекпоинта на уровне блока. |
| **Абстрактные узлы-задачи** | Backbone, Injector, … | Тип узла-задачи, конфиг экземпляра, порты. | Веса узла-задачи = state_dict блока; сериализация и загрузка на уровне узла. |
| **Гиперграф задачи** | Гиперграф | nodes, edges, exposed_inputs, exposed_outputs, schema_version, graph_id, metadata ([03_TASK_HYPERGRAPH.md](03_TASK_HYPERGRAPH.md) §9.1). | Словарь node_id → state_dict (или block_id при общем блоке). **Загрузка гиперграфа из чекпоинта** — восстановление структуры из конфига + загрузка весов всех узлов. |
| **Воркфлоу** | Гиперграф гиперграфов | graphs, edges, exposed_inputs, exposed_outputs, schema_version, workflow_id, metadata ([04_WORKFLOW.md](04_WORKFLOW.md) §8.1). | Агрегат чекпоинтов всех гиперграфов-узлов (graph_id → чекпоинт). **Загрузка воркфлоу из чекпоинта** — структура из конфига + загрузка чекпоинтов в каждый гиперграф-узел. |
| **Стадия** | Гиперграф воркфлоу | Узлы-воркфлоу, гиперрёбра, state_input_map, state_output_map, condition, schema_version, stage_id, metadata ([05_STAGE.md](05_STAGE.md)). | Агрегат чекпоинтов всех воркфлоу-узлов. Загрузка стадии из чекпоинта — структура + чекпоинты воркфлоу. |
| **Мир** | Гиперграф стадий | Два поля: **(1) исполняемая часть** — структура мира (стадии, цикл, state_schema, порядок, конфиги стадий и вложенных воркфлоу/гиперграфов); **(2) контент мира** — история, глоссарий, персонажи, сюжет ([06_WORLD.md](06_WORLD.md) §9). | Чекпоинт мира = агрегат чекпоинтов всех стадий (и вложенных воркфлоу/гиперграфов). Загрузка мира из чекпоинта — восстановление структуры из конфига исполняемой части + контент + загрузка чекпоинтов в стадии. |
| **Вселенная** | Гиперграф миров | worlds (world_id → конфиг мира или ref), edges, ether, order/topology, universe_id, schema_version, metadata ([07_UNIVERSE.md](07_UNIVERSE.md) §11.1). | **Чекпоинт вселенной** = совокупность чекпоинтов всех **развивающихся** миров (world_id → чекпоинт мира). End World не имеют исполняемого чекпоинта; их контент входит в сериализацию контента. Загрузка вселенной из чекпоинта — структура из конфига + загрузка чекпоинтов в развивающиеся миры. |

## 3. Загрузка из чекпоинта на каждом уровне

Для **каждого уровня** реализация должна поддерживать **загрузку из чекпоинта**:

- **Блок:** `load_state_dict(state_dict)` или загрузка из файла — восстановление весов блока.
- **Узел:** загрузка весов блока, размещённого в узле.
- **Гиперграф задачи:** загрузка из чекпоинта — восстановление структуры (from_config) и загрузка state_dict в каждый блок по node_id (или block_id при общем блоке). Один файл чекпоинта или директория по node_id.
- **Воркфлоу:** загрузка из чекпоинта — структура из конфига + загрузка чекпоинта каждого гиперграфа-узла. Возможна **загрузка только чекпоинтов** в уже собранный воркфлоу (без перечтения конфига).
- **Стадия:** загрузка из чекпоинта — структура + чекпоинты всех воркфлоу-узлов.
- **Мир:** загрузка из чекпоинта — конфиг исполняемой части + контент мира + чекпоинты всех стадий (и вложенных сущностей). Полное восстановление мира (структура цикла, стадии, state_schema + все веса).
- **Вселенная:** загрузка из чекпоинта — конфиг вселенной + чекпоинты всех развивающихся миров + опционально снапшот эфира. Возможна загрузка только чекпоинтов в уже собранную вселенную.

## 4. Особые случаи

### 4.1 Мир: два поля сериализации

Сериализация **развивающегося мира** разделена на **два крупных поля** ([06_WORLD.md](06_WORLD.md) §9.1):

1. **Исполняемая часть:** конфиг структуры мира (стадии, цикл, state_schema, конфиги стадий, воркфлоу, гиперграфов) + чекпоинт мира (агрегат чекпоинтов стадий и вложенных уровней).
2. **Контент мира:** история, глоссарий, персонажи, сюжет, артефакты — всё, что составляет «содержимое» мира и обновляется при World update.

При сохранении мира сохраняются оба поля. При загрузке восстанавливаются структура, веса и контент. Контент может храниться отдельно (отдельный файл или хранилище) по соглашению реализации.

### 4.2 End World

**End World** — мир, который **не исполняется** и **не изменяется**. В сериализации присутствует **только контент мира** (история, глоссарий, персонажи, сюжет); **исполняемой части и чекпоинта нет**. Загрузка End World — загрузка только контента. В конфиге вселенной End World задаётся типом `end_world` и конфигом/путём контента ([07_UNIVERSE.md](07_UNIVERSE.md) §3.2, §11).

### 4.3 Вселенная: агрегат чекпоинтов

Вселенная **не имеет своего единого state_dict**. Исполняемое состояние распределено по **развивающимся** мирам. Чекпоинт вселенной — словарь `world_id → чекпоинт мира` только для развивающихся миров. По соглашению реализация предоставляет **state_dict()** (агрегат state_dict развивающихся миров) и **load_state_dict(state)** для совместимости со стадией и развёртыванием ([07_UNIVERSE.md](07_UNIVERSE.md) §11.2).

### 4.4 Общий блок и дедупликация

Если несколько узлов гиперграфа разделяют **один и тот же блок** (один block_id, один экземпляр в памяти), в чекпоинте хранится **одна** запись по block_id; при загрузке один и тот же state_dict передаётся в единственный экземпляр блока. Аналогично на уровне воркфлоу: общий ref на один гиперграф — один экземпляр, один чекпоинт. Это обеспечивает отсутствие дублирования весов в памяти и при сохранении ([03_TASK_HYPERGRAPH.md](03_TASK_HYPERGRAPH.md) §9.2, §12.6).

## 5. Повторное использование одной модели

**Повторное использование одной модели** — сценарий, когда один и тот же блок (одна и та же архитектура и чекпоинт) используется **в нескольких узлах** одного гиперграфа или в нескольких гиперграфах/воркфлоу. Цель — экономия памяти и хранения, единый источник весов для одной «модели», разные роли в графе за счёт разных связей и контекста.

### 5.1 На уровне гиперграфа задачи

- **Один block_id на несколько узлов:** несколько узлов гиперграфа ссылаются на **один и тот же блок** (один block_id, один экземпляр в памяти). В чекпоинте гиперграфа хранится **одна** запись по block_id; при загрузке один state_dict загружается в единственный экземпляр блока, и все узлы с этим block_id используют этот экземпляр при run. Дедупликация весов и при сохранении, и при загрузке ([03_TASK_HYPERGRAPH.md](03_TASK_HYPERGRAPH.md) §9.2, §12.6).
- **Разные роли при одном блоке:** один и тот же блок может выступать в разных ролях (например, один и тот же энкодер как Conjector в одном месте и как Helper в другом) за счёт разных портов и связей; семантика задаётся топологией графа, а не копированием весов.

### 5.2 На уровне воркфлоу и выше

- **Общий гиперграф как узел:** в воркфлоу несколько узлов могут ссылаться на **один и тот же гиперграф** (один ref, один экземпляр). Чекпоинт воркфлоу содержит один чекпоинт для этого гиперграфа; при загрузке он загружается один раз и переиспользуется во всех узлах с этим ref.
- **Общий блок в разных гиперграфах:** один и тот же блок (тип + чекпоинт) может входить в разные гиперграфы. Каждый гиперграф при сохранении может хранить ссылку на общий чекпоинт блока (по идентификатору или пути) или копию — по соглашению реализации; при загрузке возможна загрузка одного чекпоинта и подстановка в несколько графов для экономии памяти.

### 5.3 Обучение при повторном использовании

При обучении **общий блок** (один экземпляр, несколько узлов) имеет **один набор параметров**; градиенты от всех узлов, использующих этот блок, суммируются (или агрегируются по политике) при обратном проходе. Реализация обеспечивает, что параметры не дублируются и обновление весов происходит один раз на один блок. См. [DOMAINS_DEPLOYMENT_TRAINING.md](DOMAINS_DEPLOYMENT_TRAINING.md) §7 (дедупликация параметров при общих блоках).

## 6. Следствия для реализации

- **Фундамент:** у каждого блока — сериализуемый конфиг, идентификаторы и **state_dict**; сохранение и загрузка блока из чекпоинта.
- **Гиперграф, воркфлоу, стадия, мир, вселенная:** структура задаётся конфигом (YAML/JSON); **веса сериализуются на каждом уровне** (агрегация весов вложенных сущностей); для каждого уровня — API сохранения и загрузки из чекпоинта (например, `save_checkpoint(path)`, `load_from_checkpoint(path)` или `state_dict()` / `load_state_dict()`).
- **Воспроизводимость:** конфиг + чекпоинт + входы (и опции run при необходимости) задают воспроизводимое поведение. Всё, что влияет на результат run, должно быть явно в конфиге, чекпоинте или входах ([01_FOUNDATION.md](01_FOUNDATION.md), философия проекта).

---

**Итог.** Документ задаёт **единый принцип сериализации на всех уровнях**: конфиг (структура) + чекпоинт (веса/состояние); загрузка из чекпоинта на каждом уровне; особые правила для мира (два поля), End World (только контент) и вселенной (агрегат чекпоинтов миров); **повторное использование одной модели** (общий блок/гиперграф, дедупликация весов и параметров). Реализация save/load и воспроизводимости опирается на эту спецификацию.
