# Глоссарий терминов

**Назначение:** краткий **справочник ключевых терминов** системы Иггдрасиль с отсылками к уровням и документам. Помогает быстро найти определение и контекст использования термина.

**Язык:** русский.

---

## A–B

| Термин | Краткое определение | Где подробно |
|--------|---------------------|--------------|
| **Action** | Опциональное взаимодействие пользователя с миром в начале шага (реплика, команда, выбор); передаётся в стадию среды (Development) как контекст. | [06_WORLD.md](06_WORLD.md) §4, §7 |
| **API (модели по API)** | Режим блока без локальных весов: инференс через запрос к провайдеру (LLM, VLM, генерация по API); контракт портов тот же, что при локальном выполнении. | [DOMAINS_DEPLOYMENT_TRAINING.md](DOMAINS_DEPLOYMENT_TRAINING.md) §5 |
| **Агент (узел-агент)** | Блок с состоянием и протоколом tool_calls/tool_results; движок выполняет agent_loop (вызов агента → инструменты → агент до завершения). | [DOMAINS_DEPLOYMENT_TRAINING.md](DOMAINS_DEPLOYMENT_TRAINING.md) §2; [03_TASK_HYPERGRAPH.md](03_TASK_HYPERGRAPH.md) §7.3 |
| **Block (блок)** | Abstract Base Block — минимальная единица системы: хранит параметры, веса, состояние; объявляет порты; выполняет run(inputs)→outputs. Материальное начало. | [01_FOUNDATION.md](01_FOUNDATION.md) §2 |
| **block_id** | Уникальный идентификатор экземпляра блока в графе или области видимости. | [01_FOUNDATION.md](01_FOUNDATION.md) §2.3 |
| **block_type** | Тип блока (строка или enum); определяет класс/реализацию; используется реестром для создания экземпляра. | [01_FOUNDATION.md](01_FOUNDATION.md) §2.3, §7 |

## C–E

| Термин | Краткое определение | Где подробно |
|--------|---------------------|--------------|
| **can_run** | Условие выполнения стадии: предикат (stage, state, context); при невыполнении стадия пропускается. | [05_STAGE.md](05_STAGE.md) §3.4; [06_WORLD.md](06_WORLD.md) §3.3 |
| **Конфиг** | Описание структуры сущности (узлы, рёбра, маппинги, метаданные); сериализуется в YAML/JSON; используется для from_config и воспроизводимости. | [01_FOUNDATION.md](01_FOUNDATION.md) §9; [SERIALIZATION.md](SERIALIZATION.md) |
| **Контент мира** | История, глоссарий, персонажи, сюжет, артефакты мира; обновляется при World update; хранится отдельно от исполняемой части при сериализации мира. | [06_WORLD.md](06_WORLD.md) §7, §9.1 |
| **Развёртывание на эндпоинтах** | Каждый гиперграф может быть развёрнут на своём эндпоинте; оркестрация вызовов по контракту run; поведение эквивалентно локальному запуску. | [DOMAINS_DEPLOYMENT_TRAINING.md](DOMAINS_DEPLOYMENT_TRAINING.md) §6 |
| **Development (развитие мира)** | Подшаг стадии среды: порождение **нового state** для следующего витка цикла (например, слот намерения); может выполняться безусловно; опционально учитывает Action. | [06_WORLD.md](06_WORLD.md) §7 |
| **End World** | Завершённый мир: только контент, без исполняемой части; только чтение; гиперссылки из других миров; в вселенной — узел без исходящих рёбер с записью. | [06_WORLD.md](06_WORLD.md); [07_UNIVERSE.md](07_UNIVERSE.md) §3.2, §12.1 |
| **Эфир вселенной** | Единая среда вселенной: общий контекст (глоссарий, правила), единый поток трансляции событий; доступ миров при run. | [07_UNIVERSE.md](07_UNIVERSE.md) §5 |

## G–H

| Термин | Краткое определение | Где подробно |
|--------|---------------------|--------------|
| **Гиперграф** | Структура: множество узлов + множество гиперрёбер (связи портов); на каждом исполняемом уровне — гиперграф; различаются узлы и то, что течёт по связям. | [01_FOUNDATION.md](01_FOUNDATION.md) §5; [Scheme.md](../Scheme.md) §2 |
| **Гиперребро** | Связь между портами узлов; арность 2 или больше (один источник → несколько приёмников, несколько источников → один приёмник); по ребру передаются данные, state или обмен. | [01_FOUNDATION.md](01_FOUNDATION.md) §5.1; [03_TASK_HYPERGRAPH.md](03_TASK_HYPERGRAPH.md) §3.2 |
| **Гиперссылки** | В контенте миров, вики-источниках и обмене между мирами **поддерживаются гиперссылки**: ссылки на статьи, сущности, другие миры или артефакты. Контент (история, глоссарий, вики-импорт) может содержать и сохранять гиперссылки; формат и разрешение ссылок задаются реализацией. | [06_WORLD.md](06_WORLD.md) §7; [WIKI_AND_CONTENT_SOURCES.md](WIKI_AND_CONTENT_SOURCES.md); [07_UNIVERSE.md](07_UNIVERSE.md) §12.1 |
| **Движок гиперграфа** | Единый механизм выполнения: хранилище структуры, валидатор, планировщик, буферы, исполнитель; обход узлов, передача данных по рёбрам, вызов run; применяется на уровнях 0.3–0.7. | [HYPERGRAPH_ENGINE.md](HYPERGRAPH_ENGINE.md); [03_TASK_HYPERGRAPH.md](03_TASK_HYPERGRAPH.md) §6 |

## I–N

| Термин | Краткое определение | Где подробно |
|--------|---------------------|--------------|
| **Исполняемый уровень** | Уровень, на котором определён контракт **run** и выполняется обход гиперграфа: гиперграф задачи (0.3), воркфлоу (0.4), стадия (0.5), мир (0.6), вселенная (0.7). | [Scheme.md](../Scheme.md) §2, §3 |
| **Node (узел)** | Abstract Graph Node — место в гиперграфе и связи (гиперрёбра); не хранит данные; в узле размещается ровно один блок. Идеальное начало. | [01_FOUNDATION.md](01_FOUNDATION.md) §4 |
| **node_id** | Уникальный идентификатор узла в гиперграфе (или graph_id во воркфлоу, world_id во вселенной). | [03_TASK_HYPERGRAPH.md](03_TASK_HYPERGRAPH.md) §3.1 |

## P–R

| Термин | Краткое определение | Где подробно |
|--------|---------------------|--------------|
| **payload_spec** | Спецификация полезной нагрузки гиперребра вселенной: что передаётся между мирами (state_snapshot, artifacts, context, events, entities); направление и момент синхронизации. | [07_UNIVERSE.md](07_UNIVERSE.md) §4.2 |
| **Порт** | Вход или выход блока/узла; имя + тип (или схема); в гиперрёбрах связываются порты разных узлов. | [01_FOUNDATION.md](01_FOUNDATION.md) §2.3, §2.4 |
| **Реестр типов блоков** | Отображение block_type → класс или фабрика; создание экземпляра блока по конфигу; расширяемость без изменений ядра. | [01_FOUNDATION.md](01_FOUNDATION.md) §7 |
| **run** | Контракт выполнения: run(сущность, входы, **options) → выходы. Сигнатура различается по уровню (inputs/outputs, state, world_inputs/results). | [Scheme.md](../Scheme.md) §3; по уровням: 03 §8, 04 §7, 05 §8, 06 §8, 07 §10 |

## S

| Термин | Краткое определение | Где подробно |
|--------|---------------------|--------------|
| **state** | Контейнер с именованными слотами; передаётся между стадиями мира и между мирами (снимок); претерпевает метаморфозы на стадиях. | [05_STAGE.md](05_STAGE.md) §3; [06_WORLD.md](06_WORLD.md) §3 |
| **state_input_map** | Правило маппинга слотов state на входы воркфлоу-узлов стадии при run(stage, state). | [05_STAGE.md](05_STAGE.md) §3.2 |
| **state_output_map** | Правило маппинга выходов воркфлоу-узлов стадии в слоты state после run воркфлоу. | [05_STAGE.md](05_STAGE.md) §3.3 |
| **state_schema** | Описание структуры state: перечень слотов, опционально типы/семантика; валидация и условия can_run. | [06_WORLD.md](06_WORLD.md) §3.2 |
| **Стадия** | Третий исполняемый уровень; гиперграф воркфлоу; внешний вход/выход — state; одна стадия = одна метаморфоза state. Роли в мире: автор, среда, творец. | [05_STAGE.md](05_STAGE.md); [06_WORLD.md](06_WORLD.md) §2.3 |
| **Сущность (entity)** | Объект с состоянием (персонаж, агент); при передаче между мирами помечается origin_world_id; кросс-мировое влияние. | [07_UNIVERSE.md](07_UNIVERSE.md) §4.2, §10.2 |

## T–W

| Термин | Краткое определение | Где подробно |
|--------|---------------------|--------------|
| **Три стадии мира** | Автор (артефакты → идея), среда (переход; World update + Development), творец (идея → артефакты). Порядок: автор → среда → творец → … | [05_STAGE.md](05_STAGE.md) §2.3; [06_WORLD.md](06_WORLD.md) §2.3; [Scheme.md](../Scheme.md) §6 |
| **Обучение (training)** | На каждом уровне поддерживается обучение trainable блоков/узлов/гиперграфов; конфиг задаёт trainable/frozen; дедупликация параметров при общем блоке. | [DOMAINS_DEPLOYMENT_TRAINING.md](DOMAINS_DEPLOYMENT_TRAINING.md) §7 |
| **Чекпоинт** | Сериализованное исполняемое состояние (state_dict весов, опционально кэш); вместе с конфигом обеспечивает воспроизводимость. | [SERIALIZATION.md](SERIALIZATION.md) |
| **World update** | Подшаг стадии среды: применение state к контенту мира при **полном** state; сохранение state; при неполном state не выполняется. | [06_WORLD.md](06_WORLD.md) §7 |
| **Воркфлоу** | Второй исполняемый уровень; гиперграф, узлы которого — целые гиперграфы задач; по рёбрам — данные между задачами. | [04_WORKFLOW.md](04_WORKFLOW.md) |
| **Мир (World)** | Четвёртый исполняемый уровень; гиперграф стадий; по рёбрам — state; цикл автор → среда → творец; state_schema, контент мира, End World. | [06_WORLD.md](06_WORLD.md) |
| **Вселенная (Universe)** | Пятый (последний) исполняемый уровень; гиперграф миров; по рёбрам — обмен (payload_spec); эфир; переход сущностей; граница онтологии. | [07_UNIVERSE.md](07_UNIVERSE.md) |

## Узлы-задачи (абстрактные роли)

| Термин | Краткая роль | Где подробно |
|--------|--------------|--------------|
| **Backbone** | Ядро; основная вычислительная единица задачи (например, UNet, DiT, LLM). | [02_ABSTRACT_TASK_NODES.md](02_ABSTRACT_TASK_NODES.md) §4 |
| **Injector** | Встраивается внутрь ядра; влияет на генерацию изнутри (например, LoRA). | [02_ABSTRACT_TASK_NODES.md](02_ABSTRACT_TASK_NODES.md) §5 |
| **Conjector** | Стоит рядом с ядром; влияет на генерацию условием (например, CLIP-энкодер). | [02_ABSTRACT_TASK_NODES.md](02_ABSTRACT_TASK_NODES.md) §6 |
| **Inner Module** | Встраивается внутрь цикла (например, солвер шага деноизинга — DDIM). | [02_ABSTRACT_TASK_NODES.md](02_ABSTRACT_TASK_NODES.md) §7 |
| **Outer Module** | Стоит рядом с циклом (например, генератор шума, расписание таймстепов). | [02_ABSTRACT_TASK_NODES.md](02_ABSTRACT_TASK_NODES.md) §8 |
| **Helper** | Вспомогательная функция; непокрытые роли (RAG, инструменты агента). | [02_ABSTRACT_TASK_NODES.md](02_ABSTRACT_TASK_NODES.md) §9 |
| **Converter** | Преобразование данных (токенизатор, VAE encode/decode). | [02_ABSTRACT_TASK_NODES.md](02_ABSTRACT_TASK_NODES.md) §10 |
