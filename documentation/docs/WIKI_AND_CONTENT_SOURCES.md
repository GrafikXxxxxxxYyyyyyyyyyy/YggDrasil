# Вики и внешние источники контента

**Назначение:** описание поддержки **формирования и наполнения мира (и при необходимости вселенной) по внешним источникам контента** — вики-ресурсам (Fandom.com, Википедия и аналоги). Один URL или идентификатор ресурса даёт конфигурацию или начальное наполнение мира; далее мир работает по канону. Документ задаёт место в архитектуре, требования к входу/выходу и маппингу в модель мира.

**Связь с уровнями:** [06_WORLD.md](06_WORLD.md) (контент мира, начальное наполнение, World update), [07_UNIVERSE.md](07_UNIVERSE.md) (контент миров во вселенной), [SERIALIZATION.md](SERIALIZATION.md) (сериализация контента).

**Философский якорь:** [../../philosophy/Philosophy-Part3.md](../../philosophy/Philosophy-Part3.md) §3.10 — поддержка вики-ресурсов; любой источник, из которого можно извлечь структурированный или текстовый лор, должен поддерживаться.

**Язык:** русский.

---

## 1. Назначение

Система должна позволять **сформировать или наполнить мир по ссылке на вики-ресурс**: заполнение в сериализованной форме на основе контента с сайта — описание мира, персонажи, глоссарии, истории, статьи. Поддерживаются не только Fandom.com, но и Википедия и аналогичные вики-ресурсы. **Любой источник**, из которого можно извлечь структурированный или текстовый лор, должен поддерживаться в рамках контракта. Один URL или идентификатор ресурса даёт конфигурацию или **начальное наполнение мира**; далее мир работает по канону ([06_WORLD.md](06_WORLD.md)).

Цель — быстрый старт мира из готовой базы знаний сообщества (Fandom-вики, вики-проекты) без ручного копирования: один URL → конфигурация/начальное содержание мира.

---

## 2. Вход и выход

| Элемент | Описание |
|---------|----------|
| **Вход** | URL вики (например, `https://<fandom>.fandom.com/wiki` или ссылка на подраздел вики; или страница Википедии/аналога). Опционально — лимиты обхода, глубина, фильтры. |
| **Выход** | Мир (или конфиг мира + начальное содержание), заполненный в **сериализованной форме** на основе данных с указанного источника. Контент мира: описание мира, история, глоссарий, персонажи, правила, стиль, нарратив — в формате, пригодном для сериализации мира по [06_WORLD.md](06_WORLD.md) §9 и для использования в цикле (Development, контекст для стадий). |
| **Источник данных** | Лор с вики: тексты статей, структура (категории, ссылки), описание персонажей, локаций, терминов, сюжетов — в объёме, доступном по переданной ссылке (в т.ч. обход по внутренним ссылкам вики в заданных пределах). Детали парсинга, лимитов обхода и маппинга задаются реализацией. |

---

## 3. Маппинг в модель мира

Полученные данные **маппятся в поля сериализации мира** ([06_WORLD.md](06_WORLD.md) §7, §9): **описание мира**, **начальное содержание** (initial story, глоссарий, правила, стиль), **персонажи**, **глоссарии**, **истории/нарратив** — так, чтобы мир можно было загрузить и использовать в цикле (Development of the world, контекст для стадий автора, среды, творца). Формат контента мира задаётся конфигом мира и реализацией; вики-импорт производит структуру, совместимую с этим форматом.

---

## 4. Место в архитектуре

- **Формирование мира по URL** — сценарий **инициализации/наполнения контента мира**, а не замена цикла стадий или схемы state. Структура мира (стадии, цикл, state_schema) задаётся конфигом мира как обычно; вики-источник заполняет **контент** (и при необходимости начальный конфиг мира).
- Результат импорта — **конфиг мира** и/или **начальное содержание мира** ([06_WORLD.md](06_WORLD.md) §5.2, set_initial_content / set_content). Дальше мир работает по канону: цикл автор → среда → творец, state, World update, Development.
- При необходимости мир, собранный по вики, может быть зафиксирован как **End World** ([07_UNIVERSE.md](07_UNIVERSE.md) §3.2) — только контент, без исполняемой части; или остаться развивающимся миром с циклом.

---

## 5. Требования к реализации

- Реализация обеспечивает **импорт/парсинг** или использование API источника (в рамках правил сайта и лицензий). Конкретные парсеры, лимиты обхода, кэш, этика использования и лицензии задаются реализацией и при необходимости отдельными правилами.
- Контракт **не привязан к одному сайту**: Fandom.com, Википедия и другие вики-источники с извлекаемым структурированным или текстовым лором поддерживаются через единый или расширяемый интерфейс (URL → контент мира / конфиг).
- Хранение результата импорта — по правилам сериализации мира: контент входит в поле «контент мира» ([SERIALIZATION.md](SERIALIZATION.md) §4.1); исполняемая часть мира при необходимости задаётся отдельно (конфиг стадий, state_schema и т.д.).

---

**Итог.** Документ задаёт **поддержку вики и внешних источников контента** для формирования и наполнения мира: вход (URL вики), выход (мир или конфиг + начальное содержание), маппинг в модель мира, место в архитектуре и требования к реализации. Философский принцип: возможность использовать такие ресурсы без жёсткой привязки к одному сайту.
